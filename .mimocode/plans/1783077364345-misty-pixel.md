# План: "Ща как напишу" — режим быстрого накидывания текста

## Суть

Режим, при котором выделенный текст из любого текстового блока автоматически добавляется в блок «Заметка». Быстро накидываем текст из кусочков.

## Что нужно сделать

### 1. SVG-иконка кнопки "Ща как напишу" в `blocks.js`

- **Где:** `blocks.js`, рядом с `TRANSLATE_SVG` (~строка 2089), в `renderTextBody`
- **Что:** Добавить SVG-иконку (стиль как у `translate-btn` — `font-ctrl-btn`), рядом с кнопкой «Перевести» в футере текстового блока
- **Иконка:** Плюс/剪贴板-стиль — например, `<svg>` с папкой или ножницами (подберу иконку, которая визуально отличается от «Перевести»)
- **Вставка:** `footer.insertBefore(captureBtn, translateBtn)` — перед кнопкой «Перевести»

### 2. Состояние режима в `blocks.js`

- **Глобальное состояние:** `let _captureMode = false` и `let _captureStickyId = null` — внутри IIFE `Blocks`
- При активации ищем/создаём sticky-блок, сохраняем его `id` в `_captureStickyId`
- Кнопка при активации: `classList.add('active')`, SVG меняется на «активный» вариант
- Доступ к состоянию: `_captureMode` — проверяется при `mouseup` на всех textarea

### 3. Обработчик выделения текста — `mouseup` listener

- **Где:** Внутри `renderTextBody`, на `ta` (textarea текстового блока)
- **Логика:**
  1. Проверяем `_captureMode === true`
  2. Получаем выделение: `ta.value.slice(ta.selectionStart, ta.selectionEnd)`
  3. Если выделение пустое — выходим
  4. Находим sticky-блок по `_captureStickyId` из `State.getActive().blocks`
  5. Если sticky не найден — создаём через `State.addBlock('sticky')`, обновляем `_captureStickyId`
  6. «Умное копирование»:
     - Если `b.value` пустой или заканчивается на `\n\n` — просто добавляем текст
     - Иначе — добавляем `\n\n` + текст (пустая строка между выделениями)
  7. Обновляем `b.value`, вызываем `State.updateLive()` для мгновенного обновления textarea в DOM
  8. Если sticky-блок collapsed — разворачиваем его

### 4. Обновление textarea заметки в DOM (без полного re-render)

- **Проблема:** `Blocks.render()` пересоздаёт весь DOM, что сбросит фокус/курсор
- **Решение:** Находим textarea sticky-блока по `data-id` и обновляем `value` напрямую:
  ```js
  const stickyEl = document.querySelector(`.block[data-id="${_captureStickyId}"] .block-textarea`);
  if (stickyEl) { stickyEl.value = newvalue; autoGrow(stickyEl); }
  ```
- Если sticky не в DOM (collapsed или не добавлен) — обновляем только `b.value`

### 5. Пульсация кнопки при активном режиме — CSS в `styles.css`

- **Ключевые кадры:** 20–30 пульсаций в минуту = 2–3 секунды на цикл
  ```css
  @keyframes capturePulse {
    0%, 100% { box-shadow: 0 0 0 0 rgba(79,142,247,0); }
    50% { box-shadow: 0 0 8px 2px rgba(79,142,247,0.35); }
  }
  .font-ctrl-btn.capture-active {
    animation: capturePulse 2.5s ease-in-out infinite;
    color: var(--accent) !important;
    opacity: 1 !important;
  }
  ```

### 6. Кнопка «Копировать в буфер» на блоке «Заметка» — `blocks.js`

- **Где:** `blocks.js`, `createHeader()`, блок `else if (b.type === 'sticky')` (~строка 723)
- **Что:** Добавить кнопку `block-btn` с SVG-иконкой копирования (из `svgIcon('copy')`) рядом с кнопкой «Цвет заметки»
- **Вставка:** `h.insertBefore(copyNoteBtn, colorBtn)` — перед кнопкой цвета
- **Обработчик:** `navigator.clipboard.writeText(b.value)` с Toast-уведомлением
- **Учитываем:** флаг `window._clipboardApiEnabled` — если выключен, fallback на `execCommand('copy')`
