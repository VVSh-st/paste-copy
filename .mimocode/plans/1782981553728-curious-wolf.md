# План: Sugiyama layout с dummy nodes для flowchart.js

## Контекст

Переписываем ядро layout engine в flowchart.js. Текущий BFS-row layout не решает проблему "связи проходят сквозь карточки". Решение — полный Sugiyama pipeline с dummy nodes для длинных рёбер.

**Референс:** Auditor 4 (TICKET-flowchart-layout-core.md, Ответ 4.txt)

## Файлы

- `flowchart.js` — замена `_flowchartLayout()`, `_drawEdge()`, `_render()`
- `flowchart.css` — без изменений

---

## Архитектура (6 фаз)

```
input(nodes, edges)
  → breakCycles()        — DFS, реверс back-edges
  → assignLayers()       — longest-path layering
  → insertDummyNodes()   — dummy для длинных рёбер
  → minimizeCrossings()  — median heuristic, 8 итераций
  → assignCoordinates()  — X/Y с учётом размеров + normalisation
  → routeEdges()         — waypoints через dummy, Catmull-Rom
output(node.x, node.y, edge.path)
```

---

## Фаза 1: breakCycles

DFS с white/gray/black state. Back-edge = ребро к серому узлу. Реверсим back-edges для layering, сохраняем для routing.

```js
function _breakCycles(nodes, edges) {
  const state = new Map(); // 0=white,1=gray,2=black
  const reversed = new Set();
  const adj = new Map();
  nodes.forEach(n => adj.set(n.id, []));
  edges.forEach(e => adj.get(e.from)?.push(e));

  function dfs(u) {
    state.set(u, 1);
    for (const v of (adj.get(u) || [])) {
      if (state.get(v) === 1) reversed.add(e); // back edge
      else if (!state.has(v)) dfs(v);
    }
    state.set(u, 2);
  }
  nodes.forEach(n => { if (!state.has(n.id)) dfs(n.id); });
  return reversed;
}
```

## Фаза 2: assignLayers (longest-path)

Топосорт по DAG (без back-edges). Longest-path: layer[child] = max(layer[child], layer[parent] + 1).

## Фаза 3: insertDummyNodes

Для каждого ребра, пересекающего >1 слоя, вставляем dummy nodes в промежуточные слои. Dummy占据 track в ordering → реальные узлы расступаются.

```js
// Для ребра from→to с layer差 > 1:
// from → d1 → d2 → ... → to
// где d1..dn — dummy в слоях layer(from)+1 .. layer(to)-1
```

Dummy: `{ id: '__d0', real: false, w: 1, h: 1 }`

## Фаза 4: minimizeCrossings (median heuristic)

8 итераций (4 вниз, 4 вверх). В каждом слое сортируем узлы по медиане позиций соседей в соседнем слое.

## Фаза 5: assignCoordinates

Y = layer * LAYER_GAP (180px).
X = центрируем каждый слой, учитывая реальные w узлов + NODE_GAP (80px) + EDGE_LABEL_GAP (40px).
Нормализация: scaleX/scaleY чтобы влезло в 2000×1400.

## Фаза 6: routeEdges + _drawEdge

Для каждого ребра собираем waypoints: [src, d1, d2, ..., dst].
- 2 точки → прямая L
- 3+ точек → Catmull-Rom сглаживание
- Back-edges → дуга сбоку диаграммы

Edge labels: на первом сегменте, перпендикулярное смещение + белая подложка.

## Изменения в flowchart.js

### Удалить:
- `_flowchartLayout()` (текущий BFS)
- `_autoLayout()`
- `_forceLayout()` (оставить только для graph mode)
- Orthogonal routing логику в `_drawEdge`

### Добавить:
- `_breakCycles(nodes, edges)` → Set reversed edges
- `_assignLayers(nodes, edges)` → Map layer
- `_insertDummyNodes(layers, edges, layerMap)` → обновлённые layers + routedEdges
- `_minimizeCrossings(layers, edges)` → сортировка слоёв
- `_assignCoordinates(layers, VCW, VCH)` → node.x, node.y
- `_catmullRomPath(pts)` → SVG path string
- `_edgeLabelPos(pts)` → { x, y }
- `_routeBackEdge(from, to)` → SVG path string (дуга)

### Обновить:
- `_drawEdge(a, b, edge)` — waypoint-based вместо polygon/bezier
- `_render()` — вызов нового pipeline
- Auto-layout button — вызов нового pipeline

### Данные для routing:
`_flowchartLayout` возвращает `{ routedEdges, dummyXY, reversed }`.
`routedEdges` = [{ e, chain: [id1, id2, ...] }]
`dummyXY` = Map<id, {x, y}>
