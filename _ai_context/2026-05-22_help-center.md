# Help Center / справка приложения — 2026-05-22

## Что сделано

- Добавлен отдельный модуль `help.js` для справочного центра.
- В `index.html` подключён `help.js` перед `app.js`.
- Кнопка справки `?` создаётся динамически рядом с `#btn-copy`.
- Справка открывается и закрывается по `F2`.
- Поддержано закрытие по `Esc`, кнопке `✕` и клику по фону overlay.
- В справке есть поиск, фильтры, хоткеи, быстрый старт, сценарии, БРО-теги, LLM-раздел, troubleshooting и копируемые примеры.
- UI оформлен в современном стиле проекта, шрифт: `"Segoe UI Variable", "Segoe UI", system-ui, -apple-system, sans-serif`.

## Проверка после внедрения

Проверено через локальную загрузку `file:///E:/Cherry_studio/index.html` в браузере:

- `help.js` загружается, `window.HelpCenter` доступен.
- Кнопка `#btn-help` создаётся.
- Overlay `#help-overlay` создаётся.
- Открытие кликом по кнопке работает.
- Открытие/закрытие по `F2` работает.
- Закрытие по `Esc` работает.
- Поиск фильтрует карточки, empty-state появляется при отсутствии совпадений.
- Копирование примеров работает через fallback, ошибок в обработчиках не поймано.
- Ошибок `window.error` и `unhandledrejection` при проверке help-flow не обнаружено.

## Мелкая правка после проверки

В `help.js` дополнительно исправлены UX/accessibility-детали:

- `#btn-help` теперь сразу получает `aria-expanded="false"` при создании.
- При открытии справки фокус ставится в поле поиска синхронно и затем повторно через `requestAnimationFrame`, чтобы не теряться на `body`.
- При закрытии фокус возвращается отложенно через `setTimeout`, чтобы избежать гонки с другими глобальными обработчиками клавиатуры.

## Обновление от 2026-05-23 — TextLinter

В `help.js` добавлен отдельный блок `Локальная причёска текста` в основных функциях:

- описывает `Быстро причесать`, diff-preview и подсказки;
- подчёркивает, что код, URL, email, `{{переменные}}`, Markdown-таблицы и команды защищаются от автоправок;
- отдельно зафиксировано поведение дробных чисел вроде `6,5`, чтобы справка соответствовала исправлению в `text-linter.js`;
- тон сохранён в стиле текущего help: полезно, коротко, с лёгким юмором без цирка с конями.

Связанная правка поведения:

- `text-linter.js` больше не вставляет пробел после запятой между цифрами, поэтому `около 6-6,5 тысяч рублей` не превращается в `около 6-6, 5 тысяч рублей`;
- общий `DiffEngine` в `llm-features.js` теперь визуализирует изменения пробелов как видимые whitespace-токены (`·`, `⇥`, `↵`), чтобы схлопывание двойных/тройных пробелов было заметно в diff, а не выглядело как фокус с исчезновением носка в стиральной машине.

Следующий шаг по TextLinter:

- при живой проверке открыть diff для строки с двойными пробелами и дробным числом;
- подтвердить, что пробелы видны в diff, а `6,5` остаётся без вставленного пробела.
