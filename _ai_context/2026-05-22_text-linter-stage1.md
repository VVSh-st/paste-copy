# TextLinter stage 1 — локальная причёска текстовых блоков

Дата: 2026-05-22
Проект: `E:\Cherry_studio`

## Контекст

Пользователь попросил добавить простой программный линтер для текстовых блоков: безопасно «причёсывать» текст без LLM, не меняя смысл. Код в чат не выводить.

## Реализовано

Файлы:

- `text-linter.js` — новый модуль `window.TextLinter`.
- `blocks.js` — компактное меню причёски в header текстового блока.
- `index.html` — подключение `text-linter.js` после `llm-features.js` и рядом с текущими UI-скриптами.

Функции:

- `TextLinter.lint(text)` — локальная безопасная нормализация.
- `TextLinter.runQuick(blockId)` — быстро применить к блоку/выделению.
- `TextLinter.openPreview(blockId)` — diff-предпросмотр + подсказки.
- `TextLinter.getSettings()`, `setSetting()`, `getSettingMeta()`.

Меню на текстовом блоке:

- `🪮 Быстро причесать`.
- `👁 Показать diff и подсказки`.
- настройки: пробелы, пунктуация, сокращения, пустые строки, hints, красная строка, абзацы, Matrix-подсветка и др.

## Важные принципы

Автоматически применяются только относительно безопасные правки:

- лишние пробелы;
- пробелы перед/после очевидной пунктуации;
- пустые строки;
- нормализация CRLF;
- типографские сокращения (`т.д.` → `т. д.`, `и т.п.` → `и т. п.`, `до н.э.` → `до н. э.`).

Запятые и спорная грамматика — только подсказки, без автозамены.

## Защита

Линтер не должен портить:

- Markdown fences;
- inline code;
- `{{variables}}`;
- URL/email;
- JSON-пары;
- dotted technical tokens (`window.TextLinter`, `app.version`, `API.Response`);
- БРО-теги и служебные `/system`, `/user`, `/assistant`, `/developer`;
- indented code;
- вложенные Markdown-списки.

## Продолжение, сделанное в этом чате

- Исправлен порядок правил сокращений: длинные варианты (`и т.д.`, `в т.ч.`, `до н.э.`) обрабатываются раньше коротких.
- Исправлена нумерация строк в hints после fenced-code блоков: теперь подсказки показывают реальные строки.
- Усилена защита подсказок от попадания в placeholder-защищённые зоны.
- Защищены uppercase dotted tokens вроде `API.Response`.
- Diff-предпросмотр берёт `font-size`, `line-height`, `font-family`, `letter-spacing`, `font-weight` из редактируемой textarea.
- Добавлены A−/A+ для размера diff, значение отображается в px.
- После принятия diff состояние блока синхронизируется напрямую в `State` и сохраняется через `Storage.save(State.serialize())`, чтобы не зависеть от того, как конкретно `input` обработчик успел обновить модель.
- В `blocks.js` добавлен fallback для meta-настроек, если `text-linter.js` ещё не загружен; lint-кнопки/чекбоксы отключаются, если API модуля недоступен.

## Проверка

Проверено в браузере `file:///E:/Cherry_studio/index.html`:

- `window.TextLinter`, `window.Blocks`, `window.LLMFeatures.DiffEngine` доступны.
- Меню причёски появляется на текстовом блоке.
- Локальные lint actions не disabled при загруженном модуле.
- Preview создаёт `.text-lint-result-panel`.
- Hints показываются без автоправки.
- Diff font-size совпадает с textarea, A+/A− меняют размер.
- Accept применяет результат, убирает panel, обновляет textarea и `State`.
- Matrix accept class появляется на textarea.
- Fenced code не трогается, hints внутри fence не появляются.
- Ошибок `window.error`/`unhandledrejection` в smoke-проверке не поймано.

Примеры проверенных результатов:

- `Привет  ,это тест .Он кривой  , но {{var , x}} и https://a.b/c,d не трогай` → `Привет, это тест. Он кривой, но {{var , x}} и https://a.b/c,d не трогай`.
- `и т.д. ещё т.п. до н.э.` → `и т. д. ещё т. п. до н. э.`.
- `window.TextLinter и app.version и API.Response` не меняется.
- `а\n```js\nдумаю что нет\n```\nдумаю что да` даёт hint на строке 5, не внутри fence.

## Дополнение после команды «продолжай»

- Защищены Markdown-таблицы: строки вида `| ... |` больше не нормализуются локальной пунктуацией и пробелами.
- Исправлена статистика лишних пустых строк: теперь считает примерный объём схлопывания, а не один общий факт.
- Усилена работа осторожных правил рядом с `{{variables}}`:
  - placeholder остаётся защищённым;

  - `capitalAfterPunctuation` не считает точку внутри placeholder концом предложения.
- Проверены smoke-кейсы: пунктуация, сокращения, fenced-code, Markdown table, NBSP рядом с placeholder, заглавные после placeholder. Все актуальные проверки прошли.
