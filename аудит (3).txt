Я прочитал промпт и сам `intelligence-core.js` из сообщения. Ниже дам именно аудит с самостоятельными правками и не буду включать пункты из списка “не предлагай”.

### [1] Критично: `acceptSuggestion()` теряет ошибки action-handler’ов

**Проблема:** если `runSuggestionAction(suggestion)` выбросит исключение, `acceptSuggestion()` только снимет lock в `finally`, но не запишет `suggestion.failed` и не покажет пользователю понятный результат.

**Строки:** примерно 494-519, `acceptSuggestion()`

**Влияние:** одна падающая интеграция (`State`, `SmartSuggestions`, `ProjectGraph`, `Storage`) превращает клик по подсказке в “ничего не произошло”, а память подсказок не получает негативный сигнал.

**Патч:**
```diff
     try {
       const ok = runSuggestionAction(suggestion);
       if (ok === 'preview') {
         track('suggestion.previewed', { action: suggestion.type });
         return true;
       }
       if (ok) {
         window.UserMemory?.updateFeatureScore?.(suggestion.type, 'accepted', suggestion.contextKey);
         track('suggestion.accepted', { action: suggestion.type });
       } else {
         track('suggestion.failed', { action: suggestion.type });
       }
       return ok;
+    } catch (err) {
+      console.error('[Intelligence] suggestion action failed:', err);
+      track('suggestion.failed', { action: suggestion.type, error: err?.message || 'unknown' });
+      window.Toast?.show?.('Не удалось выполнить подсказку', 'error');
+      return false;
     } finally {
       runningSuggestionActions.delete(actionKey);
     }
```

## Итог

Нашёл 4 критичных правки, 3 UX-правки, 1 производительную и 1 на стабильность/читаемость. Общая оценка кода: модуль в целом аккуратно защищён optional chaining и локальными fallback’ами, но самые слабые места находятся на границах публичных входов (`track`, `openPreparedReport`, external `window.*`) и в обработке ошибок action-handler’ов.
