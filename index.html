Профили и поведение LLM

Здесь выбирается модель, endpoint и общие правила LLM-запросов. Для разных БРО-тегов можно назначать разные профили — приложение отправит запрос в нужную модель и подождёт её загрузку.

Для LM Studio/Ollama запросы выполняются осторожнее: локальные модели идут по одной, без ручной выгрузки предыдущей модели. Если модель тяжёлая, увеличьте таймаут профиля.

БРО-теги

БРО-тег — это команда прямо в текстовом блоке. Напишите тег отдельной строкой, при необходимости добавьте вопрос после пробела и нажмите Enter: !бро чем усилить этот промпт?. Результат появится в этом же блоке, а !сум откроет мини-чат. Свои теги работают так же, но используют ваш отдельный системный промпт.

Как пользоваться и что делают дефолтные теги
!бро вопрос — быстрый чат по строке; если выше есть !старт, берёт текст после него как контекст. !фикс текст — исправляет опечатки, пунктуацию и грамматику без переписывания смысла. !ру / !эн — переводит весь текст текущего блока/вкладки и заменяет его переводом. !сум — делает краткое резюме текста вкладки и показывает его в мини-чате. !вопрос вопрос — отвечает строго по тексту вкладки; удобно для проверки, хватает ли контекста. !план — превращает текст вкладки в короткий список практичных шагов. !режим кратко / !режим подробно — меняет стиль ответов для текущего блока. !история показывает память диалога текущего блока, !старт очищает её.

Пользовательские теги

Обычный вызов !тег добавит ответ в текущий блок. !!тег заменит весь текст текущей вкладки на ответ LLM.

Advanced-настройка

Здесь редактируются системные инструкции встроенных LLM-функций. Если не уверены — оставьте по умолчанию.

Для своих быстрых команд используйте вкладку БРО-теги. Для выбора модели, endpoint и параметров генерации используйте Профили.

Не удаляйте обязательные переменные вроде {N}, {word}, {instruction}: они подставляются приложением перед запросом.
