# AGENTS.md

## Язык и режим

- Отвечай пользователю по-русски.
- Работай в ponytail-режиме: сначала самый короткий рабочий путь, без абстракций на будущее.
- Если задача не требует кода, не делай код.

## Devibify (UI-аудит и дизайн)

При UI-задачах (аудит интерфейса, фиксы accessibility, рефакторинг компонентов, добавление фич с UI) загружай Devibify через `read`:
- Основной файл: `Skill/skills/devibify/SKILL.md`
- Справочники по необходимости из `Skill/skills/devibify/references/`
- Сканер: `Skill/skills/devibify/scripts/scan-ui-smells.py`

Devibify НЕ в available_skills — вызов через `skill` невозможен. Используй `read` вручную.

## Рабочий цикл

1. Сначала читай `HANDOFF.md` и точные файлы, которые затрагивает задача.
2. Для нетривиальных правок проверь локальные подсказки в `Skill/`, только если они реально относятся к задаче.
3. Перед изменениями проверь `git status --short` и не трогай чужие/старые изменения.
4. Меняй минимум файлов. Исправляй причину бага в общем месте, а не симптом в одном вызове.
5. После правки запускай самый дешёвый надёжный чек: обычно `node --check <file>`, `git diff --check`, а браузер только для UI/runtime багов.
6. После каждого завершённого задания делай backup-коммит в `.git`, добавляя только файлы своей правки.

## Git и мусор

- Не добавляй `*.txt`, скриншоты и архивы backup в коммиты.
- Не чисти рабочее дерево без прямой команды пользователя.
- Если `git status` уже грязный, коммить только свои файлы через явный `git add -- file1 file2`.

## Handoff

- Держи `HANDOFF.md` коротким и исполняемым: статус, решения, проверка, следующий шаг.
- Обновляй `HANDOFF.md`, когда работа прерывается или меняется долгосрочное правило проекта.
