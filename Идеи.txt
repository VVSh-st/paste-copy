Ниже — решение в логике текущего интерфейса: минимум новых паттернов, максимум повторного использования уже существующих компонентов.

# 1. Sticky Note (блок-заметка)

Идея: это не «ещё один Text», а визуально вторичный служебный слой — мысли, гипотезы, напоминания.

## Развёрнутый вид

**Header:**

`📝 Название заметки | цвет | collapse | duplicate | menu`

Порядок кнопок справа:

1. палитра (цвет)
2. свернуть
3. дублировать
4. ⋮

Внутри:

* одно textarea
* placeholder:

> Личная заметка… Не попадает в превью

Под textarea маленький muted-label:

`Local only`

Не использовать большие бейджи.

### Цвета (5)

Вместо ярких заливок — лёгкий tint только у header + border:

* Yellow
* Blue
* Green
* Purple
* Red

Например:

```css
--note-yellow: rgba(...0.12)
--note-blue: rgba(...0.12)
```

Сам текст/фон блока остаётся общей тёмной темой.

---

## Свёрнутый вид

```text
📝 Идеи для system prompt        [🟨] [42]
```

Где:

* цвет = маленький квадрат
* `[42]` = количество символов или строк

Если пустой:

```text
📝 Empty note
```

---

## Превью

Никогда не отображается.

Дополнительно:

если пользователь открывает настройки блока:

```text
☑ Exclude from preview (locked)
```

Неактивно (чтобы правило было очевидно).

---

## Взаимодействие

Drag:

* как обычный блок
* при переносе цвет сохраняется

Горячие клавиши:

* Ctrl+Enter → collapse
* Ctrl+Shift+D → duplicate

Шаблоны:

* сохраняется цвет
* содержимое сохраняется

---

## Edge cases

Пустой:

* серый placeholder
* без warning

Очень длинный текст:

После ~10 строк:

```css
max-height
overflow:auto
```

20 заметок подряд:

Ничего специального — не плодить отдельные UI-механики.

---

## CSS

Отличия от Text:

Вместо:

```css
border-left: 1px solid
```

Использовать:

```css
border-left: 4px solid var(--note-color)
background:
linear-gradient(...)
```

Hover:

Только усиление border и лёгкое свечение.

---

### Спорный момент — выбор цвета

Вариант A:

Маленькая кнопка-палитра

Вариант B:

ПКМ по header → цвет

Вариант C:

Клик по цветной полоске

Предпочтителен A.

# 2. TODO блок

Идея: поведение максимально наследует Text (20 подвкладок), чтобы не создавать новую модель.

## Развёрнутый вид

Header:

```text
☑ Tasks | <1><2><3>... | 7/12 | collapse | ⋮
```

`7/12`

=

выполнено/всего

---

Тело:

```text
☑ Сделать экспорт
☐ Добавить IndexedDB
☐ Проверить mobile UI
☑ Починить search
```

Поведение:

Enter:

```text
☐ |
```

создаёт новую строку.

Backspace на пустой строке:

удаляет её.

Drag:

появляется handle:

```text
⋮⋮ ☑ Task
```

Показывать handle только при hover.

---

## Свёрнутый вид

```text
☑ Tasks 7/12
```

Дополнительно:

если всё выполнено:

```text
☑ Tasks 12/12 ✓
```

---

## Превью

Формат:

```text
- [x] Сделать экспорт
- [ ] IndexedDB
- [ ] Проверить UI
```

Правила:

если блок выключен:

ничего не показывать

если подвкладка пустая:

игнорируется

---

## Взаимодействие

Drag:

* блок — как сейчас
* элементы внутри — drag-reorder

Горячие клавиши:

* Enter → новый пункт
* Shift+Enter → перенос строки внутри пункта
* Ctrl+↑/↓ → перемещение задачи
* Ctrl+Enter → toggle completed

Шаблоны:

сохраняются все подвкладки

---

## Edge cases

### 20 заполненных вкладок

Проблема:

уже сейчас `<12345>` может стать тесным.

Варианты:

A:

горизонтальный скролл

B:

```text
<1><2><3><4><5> ... [+15]
```

C:

dropdown

Предпочтителен B.

---

### Очень длинный пункт

После ~2 строк:

```css
line-clamp:2
```

Полный текст:

hover → tooltip

или раскрытие по двойному клику.

---

### Пустой блок

Показывать:

```text
No tasks yet
Press Enter
```

---

## CSS

Использовать существующий стиль Text:

Новые элементы:

```css
.todo-item
.todo-checkbox
.todo-handle
.todo-completed
```

Выполненное:

```css
opacity:.65;
text-decoration:line-through;
```

Hover:

```css
background:var(--hover-bg)
```

Без новых теней.

# 3. IndexedDB backup внутри Gist Sync

Идея: это не отдельный раздел приложения, а страховка Gist.

Добавить вкладку в модалку:

```text
[Sync] [History] [Backups]
```

---

## Экран Backups

Список:

```text
🗂 Auto backup
Jun 27, 15:32
Before push #126
2.4 MB

🗂 Auto backup
Jun 27, 13:18
Before push #125
2.3 MB
```

Справа:

```text
Restore
Export
⋮
```

---

Верх панели:

```text
Local backups: 12 / 30
```

Тонкий прогресс:

```text
██████░░░░
```

Не использовать проценты.

---

## Кнопки

Верх:

```text
☑ Auto backup before sync
☑ Keep max: [30]
```

Низ:

```text
Restore selected
Delete all
```

---

## Восстановление

Не мгновенное.

Модалка:

```text
Restore backup?

Current state will become a snapshot first.

[Cancel]
[Restore]
```

Чтобы не потерять текущую работу.

---

## Edge cases

Пусто:

```text
No local backups yet
Backups appear after first sync
```

Очень много:

Пагинация не нужна.

Virtual scroll после ~30.

Повреждённый backup:

```text
⚠ Cannot read backup
```

Кнопка:

```text
Delete
```

---

## CSS

Использовать существующие карточки из snapshots:

```css
.backup-item
```

Hover:

```css
transform:translateY(-1px)
```

Минимальный эффект.

# Два спорных места

### 1. Показывать ли TODO-прогресс прямо в заголовке

Вариант A:

```text
☑ Tasks 7/12
```

Вариант B:

```text
☑ Tasks ●●●○○
```

Предпочтителен A — быстрее читается.

---

### 2. Sticky Note: скрывать ли содержимое при сворачивании

Вариант A:

только заголовок

Вариант B:

```text
📝 Идеи
"Проверить..."
```

(первая строка)

Предпочтителен B — заметки часто открывают ради быстрых подсказок.
